📤 作業繳交

GitHub 帳號設定與作業繳交教學 ▶
1
建立 GitHub 帳號
前往 github.com/signup，填入 Username（建議使用真實姓名縮寫，如 thstsai）、Email 和密碼完成註冊。
💡 帳號名稱建議：學校縮寫 + 姓名縮寫，例如 ths_mingming，方便老師辨識。
2
安裝 Git
前往 git-scm.com 下載並安裝 Git for Windows，所有選項保持預設即可。安裝後開啟「Git Bash」或「命令提示字元」確認版本：
git --version # 應顯示類似：git version 2.xx.x.windows.x
3
設定 Git 個人資訊（第一次使用時）
開啟 Git Bash，輸入以下指令設定你的姓名和 Email（與 GitHub 帳號相同）：
git config --global user.name "你的姓名" git config --global user.email "你的GitHub信箱"
4
在 GitHub 建立作業 Repository
登入 GitHub 後點選右上角「+」→「New repository」，依以下設定建立：
# Repository 設定 Repository name：BSD56_Homework Visibility： Public（公開，老師才看得到） ☑ Add a README file（勾選）
📌 建立後記下你的 Repo 網址，格式為：https://github.com/你的帳號/BSD56_Homework
5
Clone（複製）Repository 到電腦
將遠端 Repository 下載到本機，之後直接在此資料夾放作業：
git clone https://github.com/你的帳號/BSD56_Homework.git cd BSD56_Homework
6
每週繳交作業流程
完成作業後，在 BSD56_Homework 資料夾內建立 Week01/、Week02/… 子資料夾，依序執行：
# 1. 將作業檔案複製到對應資料夾，例如 Week03/ # 2. 加入所有新增/修改的檔案 git add . # 3. 建立 commit（記錄說明） git commit -m "Week03: T-SQL stored procedure 作業" # 4. 推送到 GitHub git push
🔑 第一次 push 時 Git 會要求登入，選擇「在瀏覽器中登入」即可完成 GitHub 授權。
7
送出作業表單
Push 成功後，回到本頁下方的「繳交作業表單」，填入姓名、週次與你的 GitHub Repository 網址，系統會在老師的 GitHub Issues 建立一筆繳交紀錄。
✅ 成功送出後，進度追蹤頁面的週次圖示會更新為「已完成」。
1

完成每週作業

依照週次作業要求，完成程式碼或設計圖

2

上傳到 GitHub

將作業推送到你的 GitHub Repository（命名：BSD56_W{n}）

3

填寫繳交表單

填寫下方表單，提交 GitHub Repo 連結

4

等待批閱

老師會在 GitHub Issues 中留下批閱意見

💡 提示：請先在 GitHub 建立公開 Repository，命名為 BSD56_Homework，每週建立一個資料夾（如 Week01/），並將所有作業檔案放入。
