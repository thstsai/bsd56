### 1.1 使用案例圖（Use Case Diagram）

**核心技術：UML 標準繪圖、需求分析**

#### Actor 識別方法

| Actor 類型 | 識別問題 | 範例（ZCharge） | 範例（LinkOne） |
|-----------|---------|--------------|--------------|
| 主要使用者 | 誰直接操作系統？ | 車主、充電站管理員 | 一般用戶、內容創作者 |
| 後台管理者 | 誰管理平台？ | 系統管理員 | 平台管理員、審核員 |
| 外部系統 | 哪個外部服務與本系統互動？ | 支付閘道 | AI 審核系統 |
| 非人類Actor | 哪個排程或自動化會觸發功能？ | 自動帳單排程 | （無特別強調） |

#### Use Case 關係

```
<<include>>  — 主 Use Case「必定」執行子 Use Case
              例：查詢帳單 <<include>> 線上付款

<<extend>>   — 在特定條件下「可選」延伸
              例：查詢充電站 <<extend>> 查看詳情

association  — Actor 與 Use Case 的基本連線（無特殊關係）
```

### 1.2 ER 圖設計（Entity-Relationship Diagram）

**核心技術：關聯式資料庫正規化、SQL Server DDL**

#### 三大正規形式（記憶口訣：單值→完全依賴→無遞移）

| 正規形式 | 消除目標 | 具體做法 |
|---------|---------|---------|
| **1NF** | 重複群組、非原子值 | 陣列欄位獨立成資料表 |
| **2NF** | 部分函數相依（複合PK才會有）| 每欄需完全相依整個 PK |
| **3NF** | 遞移相依（A→B→C） | 可計算得出的欄位移除（費用=電量×費率） |

#### 常用資料型別速查表

| 用途 | 型別 | 補充 |
|------|------|------|
| 姓名、名稱 | `NVARCHAR(50)` | N前綴支援中文 |
| Email | `NVARCHAR(100)` | 加 `UNIQUE` |
| 密碼 | `NVARCHAR(256)` | 存 SHA-256 雜湊（64碼hex） |
| 金額 | `DECIMAL(10,2)` | 不用 FLOAT（有浮點誤差） |
| 日期時間 | `DATETIME2` | 精度比 DATETIME 高 |
| 布林 | `BIT` | 0/1，通常 `DEFAULT 1` |

#### 56分區 LinkOne 特殊設計：New_ 前綴 + 自參照

```sql
-- 所有新增資料表加 New_ 前綴（不修改既有 SHLife 資料表）
CREATE TABLE New_Comment (
    CommentID  INT           IDENTITY(1,1) PRIMARY KEY,
    PostID     INT           NOT NULL REFERENCES New_Post(PostID),
    ParentID   INT           NULL REFERENCES New_Comment(CommentID), -- 自參照=二層留言
    AuthorID   INT           NOT NULL REFERENCES Employee(EmployeeID),
    Content    NVARCHAR(2000) NOT NULL,
    CreatedAt  DATETIME2     NOT NULL DEFAULT SYSDATETIME(),
    SafetyStatus TINYINT     NOT NULL DEFAULT 0  -- 0=待審 1=通過 2=拒絕
);
```

### 1.3 T-SQL：DDL、DML、View、Stored Procedure

**核心技術：SQL 查詢、View 封裝、BULK INSERT、Stored Procedure**

#### SQL View — 封裝複雜查詢（初學者友善）

View 是將複雜 SQL 包裝成「虛擬資料表」，C# 和 Android 直接 `SELECT * FROM vw...` 即可，**不需在程式碼內寫多表 JOIN**。

```sql
-- View：員工貼文統計（封裝 JOIN + 計算）
CREATE VIEW vw_EmployeePostSummary AS
SELECT
    e.EmployeeID,
    e.Name          AS EmployeeName,
    e.Email,
    COUNT(p.PostID) AS PostCount,
    MAX(p.CreatedAt) AS LastPostAt
FROM Employee e
LEFT JOIN New_Post p ON p.EmployeeID = e.EmployeeID
GROUP BY e.EmployeeID, e.Name, e.Email;

-- View：貼文含留言數（C# DataGridView 直接綁定）
CREATE VIEW vw_PostWithCommentCount AS
SELECT
    p.PostID,
    e.Name        AS AuthorName,
    p.Content,
    p.CreatedAt,
    p.SafetyStatus,
    COUNT(c.CommentID) AS CommentCount
FROM New_Post p
JOIN  Employee e ON e.EmployeeID = p.EmployeeID
LEFT JOIN New_Comment c ON c.PostID = p.PostID AND c.ParentID IS NULL
GROUP BY p.PostID, e.Name, p.Content, p.CreatedAt, p.SafetyStatus;

-- View：貼圖購買記錄（含貼圖名稱、買家姓名）
CREATE VIEW vw_PurchaseDetail AS
SELECT
    pur.PurchaseID,
    e.Name        AS BuyerName,
    s.Name        AS StickerName,
    s.Price,
    pur.PurchasedAt
FROM New_Purchase pur
JOIN Employee    e ON e.EmployeeID = pur.EmployeeID
JOIN New_Sticker s ON s.StickerID  = pur.StickerID;
```

> **使用建議**：先建 View，C# / Android 只需 `SELECT * FROM vw_XXX WHERE ...`，大幅降低程式碼複雜度。

#### BULK INSERT — 匯入測試資料

```sql
-- 從 CSV 匯入員工資料（競賽常見：chat_0001.csv 等）
BULK INSERT Employee
FROM 'C:\Data\employees.csv'
WITH (
    FIELDTERMINATOR = ',',
    ROWTERMINATOR   = '\n',
    FIRSTROW        = 2,        -- 跳過標題列
    CODEPAGE        = '65001'   -- UTF-8
);
```

#### Stored Procedure — 封裝業務邏輯

```sql
-- 取得員工貼文清單（呼叫 View + 加入篩選條件）
CREATE PROCEDURE sp_GetEmployeePosts
    @EmployeeID INT,
    @SafetyStatus TINYINT = NULL  -- NULL = 全部；0/1/2 = 篩選狀態
AS
BEGIN
    SET NOCOUNT ON;
    SELECT * FROM vw_PostWithCommentCount
    WHERE EmployeeID = @EmployeeID
      AND (@SafetyStatus IS NULL OR SafetyStatus = @SafetyStatus)
    ORDER BY CreatedAt DESC;
END;
```

#### CRUD 操作對比（ADO.NET vs EF）

| 操作 | ADO.NET（舊） | EF LINQ（新） |
|------|-------------|--------------|
| 讀取清單 | SqlDataAdapter + DataTable | `context.Employees.ToList()` |
| 條件篩選 | SQL `WHERE` 字串 | `.Where(e => e.IsActive == true)` |
| 新增 | `INSERT INTO ...` + ExecuteNonQuery | `context.Add(obj); context.SaveChanges()` |
| 修改 | `UPDATE ... SET ...` + ExecuteNonQuery | 直接修改物件屬性 + `SaveChanges()` |
| 刪除 | `DELETE FROM ...` + ExecuteNonQuery | `context.Remove(obj); context.SaveChanges()` |

### 2.2 可繼承基礎表單 + User Control

**設計理念：將所有 CRUD 表單的共用版型提取成 `BaseDetailForm`，各資料表對應表單繼承它，只需覆寫資料相關方法，UI 自動就位。**

#### BaseDetailForm — 可繼承 CRUD 版型

```
┌─────────────────────────────────────────┐
│  [搜尋欄]  [🔍搜尋]  [➕新增]  [✏️編輯]  [🗑️刪除]  [↻重整] │  ← Panel（CrudToolbar User Control）
├─────────────────────────────────────────┤
│                                         │
│              DataGridView               │  ← 自動填滿
│                                         │
└─────────────────────────────────────────┘
```

```csharp
// BaseDetailForm.cs — 放在 Base/ 資料夾
public partial class BaseDetailForm : Form
{
    protected DataGridView dgvMain = new DataGridView();
    protected CrudToolbar  toolbar  = new CrudToolbar();   // User Control（見下方）

    public BaseDetailForm()
    {
        InitializeLayout();
        toolbar.OnAdd     += (s, e) => OnAdd();
        toolbar.OnEdit    += (s, e) => OnEdit();
        toolbar.OnDelete  += (s, e) => OnDelete();
        toolbar.OnRefresh += (s, e) => LoadData();
        toolbar.OnSearch  += (s, e) => LoadData(toolbar.Keyword);
    }

    private void InitializeLayout()
    {
        this.Controls.Add(toolbar);
        toolbar.Dock = DockStyle.Top;

        dgvMain.Dock = DockStyle.Fill;
        dgvMain.ReadOnly = true;
        dgvMain.SelectionMode = DataGridViewSelectionMode.FullRowSelect;
        dgvMain.AllowUserToAddRows = false;
        dgvMain.AutoSizeColumnsMode = DataGridViewAutoSizeColumnsMode.Fill;
        this.Controls.Add(dgvMain);
    }

    // 子類別覆寫這三個方法即可
    protected virtual void LoadData(string keyword = "") { }
    protected virtual void OnAdd()    { }
    protected virtual void OnEdit()   { }
    protected virtual void OnDelete() { }

    protected int GetSelectedId(string idColumn)
    {
        if (dgvMain.CurrentRow == null) return 0;
        return Convert.ToInt32(dgvMain.CurrentRow.Cells[idColumn].Value);
    }
}
```

### 2.3 身份驗證與安全性

**核心技術：SHA-256 雜湊、靜態 Session、Timer 自動登出**

```csharp
// 密碼雜湊工具
public static class Security
{
    public static string HashPassword(string password)
    {
        using var sha = SHA256.Create();
        return Convert.ToHexString(
            sha.ComputeHash(Encoding.UTF8.GetBytes(password))
        ).ToLower();
    }
}

// 全域登入狀態（靜態類別取代 Session）
public static class AppState
{
    public static int    CurrentUserID   { get; set; }
    public static string CurrentUserName { get; set; } = "";
    public static bool   IsAdmin         { get; set; }
}

// 登入驗證（使用 EF）
private void Login(string email, string password)
{
    using var db = AppDb.Context;
    string hash = Security.HashPassword(password);
    var emp = db.Employees.FirstOrDefault(
        e => e.Email == email.ToLower() && e.PasswordHash == hash && e.IsActive == true);

    if (emp != null)
    {
        AppState.CurrentUserID   = emp.EmployeeID;
        AppState.CurrentUserName = emp.Name;
        new frmMain().Show();
        this.Hide();
    }
    else MessageBox.Show("帳號或密碼錯誤", "登入失敗", MessageBoxButtons.OK, MessageBoxIcon.Error);
}
```

### 2.4 內容審核（LinkOne 56分區）

```csharp
public class ContentFilter
{
    private readonly HashSet<string> _keywords;

    public ContentFilter(params string[] wordFiles)
    {
        _keywords = new HashSet<string>(StringComparer.OrdinalIgnoreCase);
        foreach (var path in wordFiles.Where(File.Exists))
            foreach (var line in File.ReadAllLines(path, Encoding.UTF8))
                if (!string.IsNullOrWhiteSpace(line))
                    _keywords.Add(line.Trim());
    }

    public bool HasBannedWord(string content) =>
        _keywords.Any(kw => content.Contains(kw, StringComparison.OrdinalIgnoreCase));

    public string GetFirstMatch(string content) =>
        _keywords.FirstOrDefault(kw =>
            content.Contains(kw, StringComparison.OrdinalIgnoreCase)) ?? "";
}
```

```kotlin
class FeedViewModel : ViewModel() {
    private val _posts     = MutableStateFlow<List<Post>>(emptyList())
    private val _isLoading = MutableStateFlow(false)
    private val _error     = MutableStateFlow<String?>(null)

    val posts:     StateFlow<List<Post>> = _posts.asStateFlow()
    val isLoading: StateFlow<Boolean>   = _isLoading.asStateFlow()
    val error:     StateFlow<String?>   = _error.asStateFlow()

    fun loadPosts() {
        viewModelScope.launch {
            _isLoading.value = true
            _error.value = null
            try {
                _posts.value = ApiClient.getPosts()
            } catch (e: Exception) {
                _error.value = "載入失敗：${e.message}"
            } finally {
                _isLoading.value = false
            }
        }
    }
}
```

### 3.4 REST API — 內建 HttpURLConnection + org.json

> **不需要 Retrofit 等外部套件**，使用 Android 內建類別即可完成 GET / POST / DELETE。

```kotlin
// ApiClient.kt — 統一管理所有 API 呼叫
object ApiClient {
    // 模擬器連本機 Server（10.0.2.2 = localhost）
    private const val BASE = "http://10.0.2.2:5000/api"

    // ── GET 請求 ─────────────────────────────────────────────
    private fun get(path: String): JSONArray {
        val conn = URL("$BASE$path").openConnection() as HttpURLConnection
        conn.requestMethod = "GET"
        conn.setRequestProperty("Content-Type", "application/json")
        conn.connectTimeout = 10_000
        conn.readTimeout    = 10_000
        return try {
            JSONArray(conn.inputStream.bufferedReader().readText())
        } finally {
            conn.disconnect()
        }
    }

    // ── POST 請求 ────────────────────────────────────────────
    private fun post(path: String, body: JSONObject): JSONObject {
        val conn = URL("$BASE$path").openConnection() as HttpURLConnection
        conn.requestMethod = "POST"
        conn.setRequestProperty("Content-Type", "application/json")
        conn.doOutput = true
        conn.connectTimeout = 10_000
        conn.outputStream.bufferedWriter().use { it.write(body.toString()) }
        return try {
            JSONObject(conn.inputStream.bufferedReader().readText())
        } finally {
            conn.disconnect()
        }
    }

    // ── DELETE 請求 ──────────────────────────────────────────
    private fun delete(path: String): Boolean {
        val conn = URL("$BASE$path").openConnection() as HttpURLConnection
        conn.requestMethod = "DELETE"
        conn.connectTimeout = 10_000
        return try {
            conn.responseCode in 200..299
        } finally {
            conn.disconnect()
        }
    }

    // ── 具體 API 方法（在 IO 執行緒呼叫）──────────────────────
    suspend fun getPosts(): List<Post> = withContext(Dispatchers.IO) {
        val arr = get("/posts")
        List(arr.length()) { i ->
            val obj = arr.getJSONObject(i)
            Post(
                id           = obj.getInt("id"),
                content      = obj.getString("content"),
                authorName   = obj.getString("author_name"),
                createdAt    = obj.getString("created_at"),
                commentCount = obj.optInt("comment_count", 0)
            )
        }
    }

    suspend fun getComments(postId: Int): List<Comment> = withContext(Dispatchers.IO) {
        val arr = get("/posts/$postId/comments")
        // 先取出所有留言，再組裝二層結構
        val all = List(arr.length()) { i ->
            val obj = arr.getJSONObject(i)
            Comment(
                id       = obj.getInt("id"),
                content  = obj.getString("content"),
                authorName = obj.getString("author_name"),
                parentId = if (obj.isNull("parent_id")) null else obj.getInt("parent_id")
            )
        }
        // 組裝：找出一層留言，把二層回覆掛上去
        val roots = all.filter { it.parentId == null }
        roots.map { root ->
            root.copy(replies = all.filter { it.parentId == root.id })
        }
    }

    suspend fun createPost(content: String, employeeId: Int): Boolean = withContext(Dispatchers.IO) {
        try {
            post("/posts", JSONObject().apply {
                put("content", content)
                put("employee_id", employeeId)
            })
            true
        } catch (e: Exception) { false }
    }

    suspend fun getStickers(): List<Sticker> = withContext(Dispatchers.IO) {
        val arr = get("/stickers")
        List(arr.length()) { i ->
            val obj = arr.getJSONObject(i)
            Sticker(
                id       = obj.getInt("id"),
                name     = obj.getString("name"),
                imageUrl = obj.getString("image_url"),
                price    = obj.getDouble("price")
            )
        }
    }

    suspend fun purchaseSticker(employeeId: Int, stickerId: Int): Boolean = withContext(Dispatchers.IO) {
        try {
            post("/purchases", JSONObject().apply {
                put("employee_id", employeeId)
                put("sticker_id",  stickerId)
            })
            true
        } catch (e: Exception) { false }
    }
}
```

| 時段 | 模組 | 工作項目 |

### 入門選手優先順序

```
必做（穩定得分）：
  ✅ Use Case 圖          — 格式正確 5 分
  ✅ 資料字典             — 逐欄填寫 8 分
  ✅ SQL View × 2         — 供 C# 和 Android 使用
  ✅ EF Scaffold + 登入   — 基礎建設 5 分
  ✅ CRUD（繼承 BaseForm）— 每個完整 CRUD ≈ 5 分
  ✅ Compose 清單頁       — LazyColumn 顯示資料 8 分

有時間才做：
  ⚡ ER 圖精修（3NF 正確 15 分）
  ⚡ C# 特殊功能（審核/詞庫/報表）
  ⚡ Android API 寫入（POST/DELETE）
  ⚡ Android 特殊功能（貼圖商城/遊戲）
```

### 技術選型總結

| 模組 | 建議技術 | 避免 | 原因 |
|------|---------|------|------|
| SA&DB | SQL View + SP | 不用 View，把 JOIN 寫在程式裡 | View 一次寫好，程式碼清爽 |
| Software | EF Database-First + BaseDetailForm | 每個表單重複寫 SqlConnection | LINQ 比手寫 SQL 字串少錯 |
| App | Jetpack Compose + 內建 HTTP | XML + Retrofit（多一層學習） | Compose 宣告式直覺，內建 HTTP 無 dependency 問題 |

### 通用黃金原則

> **完整的簡單功能 > 不完整的複雜功能**
>
> 任何功能只做一半（介面有但無資料、按鈕有但無動作）都不會得分。
> 寧可跳過困難功能，把時間用在做完容易功能。
